# Turbo Comb
A toolbox for correcting dual comb spectroscopy data.

This code demonstrates the algorithm described in
"Generalized method for the computational phase correction of arbitrary dual comb signals" by Burghoff et al. (D. Burghoff, N. Han, and J. H. Shin, "Generalized method for the computational phase correction of arbitrary dual comb signals," Opt. Lett., vol. 44, no. 12, pp. 2966–2969, June 2019, https://dx.doi.org/10.1364/ol.44.002966).

If you find this useful in your research, please cite our work!

## Quick start

Run any of the demo scripts in the `demo/` folder (they set up their own paths,
so they work from any working directory):

| Demo | What it does |
|------|--------------|
| `demo/Demo_Augmented_Kalman.m`    | Generates synthetic dual-comb data and corrects it. Reproduces the residual plot of Fig. 1 (residuals on the coherent limit). |
| `demo/Demo_Augmented_Kalman_EM.m` | Same data, but starts from an arbitrary noise guess and *learns* the noise parameters by expectation maximization. |
| `demo/Demo_MIR_QCL_DCS.m`         | Corrects a real mid-IR quantum-cascade-laser dual-comb measurement (`data/MIR_QCL_DCS.*`). |
| `demo/Demo_THz_QCL_DCS.m`         | Corrects real THz quantum-cascade-laser multiheterodyne data (`data/THz_QCL_DCS.mat`). |

`demo/Demo_Augmented_Kalman.m` does the following:

1. Generates artificial data similar to Fig. 1 (`demo/Generate_Data.m`).
2. Corrects it by running the main filter (`Augmented_Kalman.m`).
3. Generates plots assessing the efficacy of the correction (`demo/Efficacy_Plots.m`).

## Parameters (`AK_Params.m`)

Configuration is generated by `AK_Params()` and overridden field-by-field. The
key fields:

- `fD_range` — two-element vector bounding the repetition-rate difference. Most
  easily read off `abs(fft(sn)).^2`.
- `Q` — 4×4 process-noise matrix in the order `[phi_0, phi_r, f_0, f_r]`. The two
  phases have units of radians; the two frequencies have units of
  `(rep rate)^2/(rep period)`, so `Q(f0,f0)=1` means the offset fluctuates by one
  `f_r` per `T_r`. If unknown, set to `diag([1,.01,1,.01].^2)` and run EM.
- `excess_noise` — multiplier on the white-noise floor (estimated from the top
  10% of the frequency range) that sets the measurement-noise sigma. If unknown,
  set to `1` and run EM. The optimal value is usually larger than 1.
- `post_regularization` — how the smoothed phase trajectory is regularized:
  - `'lpf'` *(default)* — rubber-band band-limited filter (Xiao & Burghoff,
    Opt. Lett. 2025): a weighted least-squares fit in the passband with optimal
    padding, so the phase is low-pass filtered **without edge artifacts**. Used
    with uniform valid-region weighting, this puts the synthetic-demo residuals
    on the **coherent limit**.
  - `'ekf'` — one-step extended Kalman filter regularization. Preferable for real
    data; used by `Demo_MIR_QCL_DCS` and `Demo_THz_QCL_DCS`.
- `EM` — `1` for batch EM, `'ekf'` for one-step-EKF EM, `0` to disable.

`Augmented_Kalman_EM.m` wraps the filter in an EM loop, re-estimating `Q` and
`excess_noise` each iteration and plotting the convergence and the
before/after corrected spectra.

### A note on the rubber-band filter

The `'lpf'` regularizer is the rubber-band filter, the modern successor to the
power-weighted `Weighted_LPF` smoother of the original 2019 implementation.

## Layout

```
Augmented_Kalman.m         main correction filter
Augmented_Kalman_EM.m      EM loop wrapper
AK_Params.m                default parameters
Correct_Signal.m           applies a phase/timebase correction and computes PSDs
demo/                      the demos, plus their helpers:
  Demo_*.m                   the demos listed above
  Generate_Data.m            synthetic data generator
  Efficacy_Plots.m           efficacy + residual plots for the synthetic demo
dependencies/              helper functions (filters, plotting, FFT utilities, I/O)
data/                      truncated real-data subsets used by the demos
```

## Data

The real-data demos use *truncated* subsets so the repository stays light:

- `data/MIR_QCL_DCS.{bin,mat}` — the first Picoscope block (~8 MB) of the
  mid-IR QCL recording.
- `data/THz_QCL_DCS.mat` — the first 2 ms window of the THz QCL I/Q channels
  (`C`, `D`).

These windows cover what the demos process. Full-length recordings are not
distributed with the repository.
